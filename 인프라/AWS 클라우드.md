### VPC(Virtual Private Cloud)
- VPC는 AWS 내에서 `사용자가 정의한 가상 네트워크`로, 독립적인 IP 주소 범위를 가지고 있다. 다른 AWS 서비스와 격리된 네트워크 환경을 제공한다. VPC 안에서 서브넷, 라우팅, 보안 그룹을 설정하여 네트워크를 설계할 수 있다. 
- 목적: 애플리케이션의 네트워크 환경을 정의하고, 보안 및 접근 제어를 적용한다. 
### 서브넷(Subnet)
- VPC 내에서 IP 주소를 세분화한 네트워크 세그먼트. 퍼블릭 서브넷과 프라이빗 서브넷으로 나누어 사용할 수 있다. 
- 퍼블릭 서브넷: 인터넷 게이트웨이에 연결되어 있어 외부 인터넷에서 접근이 가능하다. 
- 프라이빗 서브넷: 인터넷 게이트웨이에 연결되어 있지 않아, 주로 백엔드 서버, 데이터 서버 등 외부에 직접 노출되지 않아야 하는 리소스를 배치한다. 
- 목적: 서브넷을 사용하면 리소스를 물리적으로 또는 논리적으로 그룹화하여 보안을 강화하고 네트워크 트래픽을 관리할 수 있다.
### AZ(Availability Zone)
- 가용 영역은 AWS 리전 내에서 물리적으로 분리된 독립적인 데이터 센터이다. 리전내에는 여러 AZ가 있으며, AZ들은 서로 독립적 전력, 냉각, 네트워크 시설을 갖추고 있다. 
- 목적: 가용성을 높이기 위해 여러 AZ에 인프라를 분산하여 배포함으로써 한 AZ에 장애가 생기더라도, 다른 AZ의 리소스가 정상적으로 동작할 수 있도록 한다. 
### 인터넷 게이트웨이
- VPC를 인터넷에 연결할 수 있게 해주는 구성 요소이다. VPC 내의 퍼블릭 서브넷에서 실행되는 리소스가 인터넷에 액세스할 수 있도록 한다. 
- 목적: 외부 인터넷에서 VPC 내의 리소스에 접근할 수 있게 하거나, VPC 내의 리소스가 인터넷에 접근할 수 있게 설정한다. 
### NAT(Network Address Translation) 게이트웨이
- NAT 게이트웨이는 프라이빗 서브넷의 인스턴스가 인터넷에 액세스할 수 있게 해주는 AWS 관리형 서비스이다. NAT 게이트웨이를 사용하면 프라이빗 서브넷에 위치한 리소스가 인터넷으로부터 직접적으로 접근받지 않으면서도 외부에 요청을 보낼 수 있다. 
- 목적: 프라이빗 서브넷의 인스턴스들이 소프트웨어 업데이트나 외부 API 요청 등을 위해 인터넷에 접근해야 하는 경우에 NAT 게이트웨이를 사용한다.

### 라우팅 테이블
- 네트워크 트래픽이 목적지에 도달할 수 있도록 경로를 정의하는 설정이다. VPC 내의 서브넷은 각기 다른 라우팅 테이블을 참조할 수 있다. 
- 목적: 라우팅 테이블을 통해 서브넷의 트래픽을 어디로 보낼지 정의한다. 예를 들어, 퍼블릭 서브넷은 인터넷 게이트웨이로 라우팅하고, 프라이빗 서브넷은 NAT 게이트웨이로 라우팅한다. 
### 보안그룹 & 네트워크 ACL
- 보안그룹: AWS 리소스에 대한 가상 방화벽 역할을 한다. 인바운드 및 아웃바운드 트래픽을 제어한다. 리소스별로 보안 그룹을 적용하여 포트, 프로토콜, IP 주소 범위를 지정할 수 있다.
- 네트워크ACL: 서브넷 수준에서 트래픽을 제어하며, 인바운드 및 아웃바운드 규칙을 IP 주소와 포트별로 설정한다. 기본적으로 stateless이며, 모든 서브넷에 적용할 수 있다.
